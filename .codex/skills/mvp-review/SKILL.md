# MVPレビューSkill

実装内容をMVP（最小実用製品）の観点からレビューしてください。

以下の観点を重視してください:

- 過剰設計になっていないか
- 不要な抽象化がないか
- 早すぎる最適化をしていないか
- ファイル数が増えすぎていないか
- 不要なGemを追加していないか
- 後回しにできる機能が含まれていないか

必要に応じて、よりシンプルな実装案を提案してください。

以下を重視して最適化してください:

- 素早く公開できること
- 保守しやすいこと
- スコープを小さく保つこと
